# Skills 全局库后续方案待办

## 背景

配置管理里的 Skills 需要从“每个工具单独安装”升级为“本软件统一管理，一次安装，多工具可用”，同时不破坏用户已经通过 Claude plugin、Codex 或其它第三方方式安装的技能。

## 已确认原则

- 全局目录使用 `~/.agent-usage`，避免和其它软件目录冲突。
- Skills 全局库放在 `~/.agent-usage/skills/`。
- 用户新导入的 skill 先进入全局库，再同步到各工具。
- 从 Claude、Codex、OpenCode 等工具目录发现的已有 skill，不删除原文件。
- 发现已有 skill 后，复制一份到全局库，并让该工具后续优先使用全局库副本。
- 同步到各工具时优先使用 symlink；不支持或失败时回退 copy。
- UI 需要展示全局库状态、来源、各工具安装状态、冲突状态。

## 建议设计

1. 新增全局库扫描：扫描 `~/.agent-usage/skills/`，解析 `SKILL.md` 或兼容元数据。
2. 新增外部发现：扫描各工具 skills 目录，识别未纳入全局库的 skill。
3. 新增导入动作：将外部 skill 复制到全局库，记录来源工具和原路径。
4. 新增同步动作：为目标工具创建 symlink/copy，使其使用全局库副本。
5. 新增冲突处理：同名不同内容时保留双方，提示用户选择全局版本或工具版本。
6. 保留原工具目录内容，不做删除；必要时只对本软件创建的链接/副本进行替换。

## 待澄清

- 同名不同内容的默认策略：保留两个并提示、以工具目录覆盖全局、或以全局覆盖目标工具。
- `~/.codex/superpowers/skills` 是否作为只读来源展示，还是导入到全局库。
- 是否需要支持 marketplace/GitHub/ZIP 安装入口的第一版范围。
